JS Fetch in about:blank Frame

Tests whether disabling JavaScript at the top level also disables it inside an about:blank iframe

What this tests

The parent page uses JavaScript to create an about:blank iframe and write a script into it. That script:

  1. Fetches /local-frame-js-fetch/data.json from the same origin.
  2. Sums the numbers in the response and multiplies by a constant: (3+7+11+19+23)×4 = 252.
  3. Writes the computed result to the frame’s document.

Because an about:blank frame inherits its security origin from the parent, a browser’s script-blocking policy should apply equally to both contexts.

To test: click Run Test with JavaScript enabled to see the frame output, then disable JavaScript for this page, reload, and observe what happens. If the frame still displays “Result: 252” with JS disabled, the browser’s JS policy is not correctly propagated to about:blank frames.
