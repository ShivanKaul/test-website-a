JS Fetch in Local Frame

Tests whether disabling JavaScript at the top level also disables it inside a local iframe document

What this tests

This page creates an iframe directly in HTML and runs a script inside it. That script:

  1. Fetches /local-frame-js-fetch/data.json from the same origin.
  2. Sums the numbers in the response and multiplies by a constant: (3+7+11+19+23)×4 = 252.
  3. Writes the computed result to the frame’s document.

Because this iframe document is local to the embedding page, a browser’s script-blocking policy should apply equally to both contexts.

To test: load this page with JavaScript enabled to see frame output, then disable JavaScript for this page, reload, and observe what happens. If the frame still displays “Result: 252” with JS disabled, the browser’s JS policy is not correctly propagated to local iframe documents.

The test runs automatically when this page loads.

Scenario Frame output Verdict
