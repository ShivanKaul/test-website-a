JS Execution in about:blank Frame

Tests whether disabling JavaScript at the top level also disables it inside a local about:blank iframe document

What this tests

When you press the button, this page creates an about:blank iframe and injects a script into it. That script:

  1. Runs entirely inside the local frame, with no network request.
  2. Computes a value in JavaScript: (3+7+11+19+23)×4 = 252.
  3. Writes the computed result to the frame’s document.

Because this iframe document is local to the embedding page, a browser’s script-blocking policy should apply equally to both contexts.

To test: with JavaScript enabled, press the button and confirm the frame shows “Result: 252”. Then disable JavaScript for this page, reload, and press the button again. If the frame still displays “Result: 252” with JS disabled, the browser’s JS policy is not correctly propagated to local iframe documents.

Press the button to run the script inside the frame.

Frame output

Expected results

Scenario Frame output Verdict
JS enabled Result: 252 Expected — JS runs in frame
JS disabled (protected) Stays empty / no computed result Pass — JS policy propagates to iframe document
JS disabled (vulnerable) Result: 252 Fail — frame JS runs despite top-level JS being off
