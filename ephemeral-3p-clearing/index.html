Ephemeral third-party storage clearing test

What this tests

This page tests whether storage written by a third-party iframe is cleared after the embedding first-party site is closed or navigated away from.

The top-level page is the first-party site. The iframe is the third-party site. The iframe writes a unique token into several storage mechanisms, then this page checks whether that token can still be read.

Expected behavior

  • Same-session phase: stored values should be present. This confirms the test can write and read third-party storage during the active first-party session.
  • After-close phase: stored values should be absent. If the same token is still present, the third-party storage partition survived after the first-party site went away.

A browser passes the ephemeral-storage claim only if storage is readable during the same first-party session and cleared after the first-party site is gone.

Before running the after-close test, close any other tabs open to this first-party origin. If another first-party tab remains open, the storage partition may legitimately remain alive.
